Available tests:

  • Reflected Javascript (injected Javascript echo):
    • xss/reflect/full1?in=change_me - Javascript echoed (Full Javascript hack). PoE: /xss/reflect/full1?in=<script>alert(/XSS/)</script>
    • xss/reflect/basic?in=2change - similar to the above, no quotes needed in the injection. Echo of unfiltered input in a "normal" HTML context (not between tags, etc.). The example shows the minimal Webseclab template consisting of just {{.In}} "moustache" placeholder. PoE: /xss/reflect/basic?in=<script>alert(/HACKED/)</script> or /xss/reflect/basic?in=<img src=foo onerror=alert(12345)>
    • xss/reflect/basic_in_tag?in=2change - similar to the above but inside of an HTML tag (still no quotes needed in the injection). Echo of unfiltered input inside of a bold tag. PoE: /xss/reflect/basic_in_tag?in=<script>alert(/HACKED/)</script> or /xss/reflect/basic_in_tag?in=<img src=foo onerror=alert(12345)>
    • xss/reflect/post1 - Javascript echoed from the POST parameters. (Splash page: xss/reflect/post1_splash)
    • xss/reflect/textarea1?in=foo1 - Javascript echoed in the textarea element (textarea closing tag required to trigger).
    • xss/reflect/textarea2?in=foo2 - Javascript echoed in the textarea element (textarea closing tag required to trigger, all tags prior to the closing textarea are filtered).
    • xss/reflect/textarea1_fp?in=fp1 - False Positive issue on injecting into the textarea block.
    • xss/reflect/textarea2_fp?in=fp2 - False Positive issue on injecting into the textarea block.
    • xss/reflect/doubq1?in=changeme2 - Double-quoted injection echoed unescaped. PoE: /xss/reflect/doubq1?in=%253Cscript%253Ealert%28%252FXSS%252F%29%253C%252Fscript%253E
    • xss/reflect/rs1?in=change_me3 - Response-Splitting (injection of \r\n\r\n into HTTP Headers making part of the headers become the body). PoE: /xss/reflect/rs1?in=xyz%0d%0a%0d%0a%3Cscript%3Ealert(/BAD_NEWS/)%3C/script%3E
    • xss/reflect/inredirect1_fp - Injection into a 302 redirect (FP check - no alert expected)
    • xss/reflect/onmouseover?in=changeme4 - XSS due to attribute injections in tags ("onmouseover").
    • xss/reflect/onmouseover_unquoted?in=changeme5 - XSS due to unquoted attribute injections in tags ("onmouseover").
    • xss/reflect/onmouseover_unquoted_fp?in=val5fp - FP of XSS due to unquoted attribute injections in tags ("onmouseover").
    • xss/reflect/onmouseover_div_unquoted?in=changeme6 - XSS due to unquoted attribute injections in a div class ("onmouseover").
    • xss/reflect/onmouseover_div_unquoted_fp?in=val6fp - FP of XSS due to unquoted attribute injections in tags ("onmouseover").
    • xss/reflect/oneclick1?in=xyz - JS injection into href source / "oneclick" XSS.
    • xss/reflect/refer1 - the Referer header echoed. You can set up a page pointing to <WEBSECLAB_URL>/misc/webseclab_refer.html?%3Cscript%3Ealert%28789%29%3C/script%3E as a starting point to set the referer.
    • xss/reflect/js3?in=js3 - exploitable injection into JS executable context (unquoted input). Proof of Exploit: /xss/reflect/js3?in=1,x:alert(12345)
    • xss/reflect/js6_sq?in=js6 - exploitable Javascript and single quotes injection into a script block. Proof of Exploit: /xss/reflect/js6_sq?in=js6%27,x:alert(12345),y:%27
    • xss/reflect/js6_sq_combo1?in=js6 - exploitable Javascript and single quotes injection into a script block. Proof of Exploit: /xss/reflect/js6_sq?in=js6%27,x:alert(12345),y:%27 Additional scanner challenge: preceeded by a non-exploitable injection into a form's input field.
    • xss/reflect/js4_dq?in=js4 - exploitable Javascript and double quotes injection into a script block. Proof of Exploit: /xss/reflect/js4_dq?in=js4%22,x:alert(12345),y:%22 .
    • xss/reflect/js_script_close?in=foo - injection of </script> tag into a quoted Javascript string. (Exploitable since browser parses script opening and closing tags first!)
    • xss/reflect/js3_fp?in=js3_fp - potential False Positive: non-exploitable injection into JavaScript fully quoted string.
    • xss/reflect/js3_notags?in=js3notags - exploitable injection into JS executable context, with HTML tags filtered out.
    • xss/reflect/js3_notags_fp?in=js3_fp - potential False Positive: non-exploitable injection into JavaScript fully quoted string, with HTML tags filtered out.
    • xss/reflect/js3_search_fp?in=js3 - potential False Positive: non-exploitable injection into JavaScript quoted string.
    • xss/reflect/js4_dq_fp?in=js4 - potential false positive: double-quotes and backslash characters are backslash-escaped resulting in valid Javascript.
    • xss/reflect/js6_sq_fp?in=js6fp - non-exploitable injection into a double-quoted property of the input tag. Check URL: /xss/reflect/js6_sq_fp?in=js6%27,x:alert(12345),y:%27
    • xss/reflect/enc2?in=enc2 - double quotes escaped with a backslash but backslash itself is not - exploitable injection into Javascript strings.
    • xss/reflect/enc2_fp?in=enc2_fp - double quotes and backslashes are escaped with a backslash - not exploitable (but still \u0022 is a better and safer way to escape double quotes).
    • xss/reflect/backslash1?in=xyz - Unicode escape sequences like \u0022 unescaped by the server.

  • Raw HTML tags issues
    • xss/reflect/raw1?in=xyz - raw HTML tags echoed in HTML text context (IE-related). Check URL: /xss/reflect/raw1?in=<xss>
    • xss/reflect/raw1_fp?in=xyz - raw HTML tags echoed inside of a Javascript quoted string, with quotes stripped (false positive)

  • DOM XSS
    • /xss/dom/domwrite?in=abc - DOM XSS due to document.write of location.href
    • /xss/dom/domwrite_hash?#in=xyz - DOM XSS due to document.write of location.hash
    • /xss/dom/domwrite_hash_urlstyle#/foo/bar?in=xyz - DOM XSS due to document.write of location.hash (URL-style value)
    • /xss/dom/yuinode_hash?#in=xyz - DOM XSS using YUI (location.hash)
    • /xss/dom/yuinode_hash_urlstyle#/foo/bar?in=xyz - DOM XSS using YUI (location.hash - URL-style value)
    • /xss/dom/yuinode_hash_unencoded?#in=xyz - DOM XSS using YUI (unescaped location.hash)

  • Miscellaneous scanner tests
    • /xss/reflect/full_cookies1?in=xyz - requires "awesome" in the Cookie header to access the vulnerable page, returns 400 Forbidden otherwise
    • /xss/reflect/full_headers1?in=xyz - requires HTTP Header "X-Letmein: 1" to access the vulnerable page, returns 400 Forbidden otherwise
    • /xss/reflect/full_useragent1?in=xyz - requires User-Agent with "Mobile"" to access vulnerable page, returns 400 Forbidden otherwise
